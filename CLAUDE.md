## Project Overview

Personal portfolio website for Nazmul Islam (nazmul.dev) - a static site generated from JSON data using Handlebars templates. Deployed to GitHub Pages with custom domain.

## Build System

The site uses a custom Node.js build script that merges JSON data files and compiles a Handlebars template into static HTML.

**Build command:**
```bash
npm run build
```

**What it does:**
- Loads all JSON files from `data/` directory (meta, profile, experience, skills, projects, contact)
- Merges data into a single object
- Compiles `src/index.hbs` Handlebars template with the merged data
- Outputs `dist/index.html`
- Copies static assets (`css/`, `images/`, `scripts/`, `resources/`) to `dist/`
- Copies `CNAME` file to `dist/` for GitHub Pages custom domain

**Preview locally:**
```bash
npx serve dist
```

## Architecture

### Data-Driven Content

All website content is stored in JSON files under `data/`:
- `meta.json` - Site metadata (title, description, navigation)
- `profile.json` - About section, bio, social links
- `experience.json` - Work experience entries
- `skills.json` - Technical skills
- `projects.json` - Portfolio projects with descriptions
- `contact.json` - Contact form configuration

**To update content:** Edit the relevant JSON file and rebuild. Do not edit `dist/index.html` directly.

### Build Process

The build script (`scripts/build.js`) performs these steps:
1. Load and parse all JSON files
2. Merge data (later files override earlier ones for duplicate keys)
3. Preprocess projects: convert `modalDescription` newlines to `<br>` tags ‚Üí `modalDescriptionHtml`
4. Compile Handlebars template with merged data
5. Write `dist/index.html`
6. Copy static assets and CNAME

### Template System

Single Handlebars template: `src/index.hbs`
- Uses merged JSON data as context
- Includes Bootstrap 4 styling and custom CSS
- Responsive design with mobile-first approach

## Deployment

**GitHub Actions workflow:** `.github/workflows/deploy.yml`

- **Auto-build:** Triggers on every push to `main` or `master` (builds artifact but does not deploy)
- **Manual deploy:** Use GitHub Actions UI ‚Üí "Deploy to GitHub Pages" ‚Üí "Run workflow" ‚Üí "Run workflow"
- **Pages setup:** In repo Settings ‚Üí Pages, set Source to "GitHub Actions"

**Custom domain:** `CNAME` file contains `nazmul.dev` and is automatically copied to `dist/` during build.

## Common Workflows

**Updating content:**
1. Edit relevant JSON file in `data/`
2. Run `npm run build`
3. Preview with `npx serve dist`
4. Commit and push (GitHub Actions will build and optionally deploy)

**Updating styles:**
1. Edit CSS files in `css/`
2. Run `npm run build` (copies CSS to dist/)
3. Preview and commit

**Updating template structure:**
1. Edit `src/index.hbs`
2. Run `npm run build`
3. Preview and commit

## Important Notes

- The `dist/` directory is gitignored and should never be committed
- Always rebuild after making any changes (content, styles, or template)
- The build script converts newlines in `projects[].modalDescription` to `<br>` tags automatically
- Static assets (CSS, JS, images, resources) are copied as-is during build
- The site uses Bootstrap 4, jQuery, and Font Awesome (loaded from CDN in template)
